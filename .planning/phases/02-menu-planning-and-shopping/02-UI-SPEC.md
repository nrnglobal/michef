## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline pill spacing |
| sm | 8px | Compact element spacing, list item internal gaps |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, card gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, empty state vertical padding |
| 3xl | 64px | Page-level spacing (cook mobile empty state) |

Exceptions:
- Shopping list check-off touch targets: minimum 48px height (SHOP-03 requirement). Use `min-h-12` (48px) on each list item row.

Source: 8-point scale (default) + REQUIREMENTS.md SHOP-03.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | List item names, card body text, form labels |
| Label | 12px (text-xs) | 400 (regular) | 1.4 | Section labels, category badges, stat captions, running count |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Page titles, modal headings |
| Display | 24px (text-2xl) | 600 (semibold) | 1.2 | Client dashboard page title only |

Notes:
- Two weights only: 400 (regular) and 600 (semibold). No medium (500) or bold (700) to maintain the restrained brand tone.
- Spanish text for the cook uses identical sizing — the i18n layer substitutes strings, typography contract is language-agnostic.
- Established by existing code: `text-2xl font-semibold` for page H1, `text-sm` for body — confirmed in `dashboard/page.tsx` and `recipes/page.tsx`.

Source: Inferred from existing codebase patterns.

Accent (`#8B6914`) reserved for:
- Primary CTA buttons (e.g. "Confirm menu", "Plan menu", "Suggest menu")
- Icon badge backgrounds use `#FEF9EC` (warm tint) with `#8B6914` icon fill
- Active tab trigger text color
- Link text color ("View all", navigation links)
- Running count accent text (checked/total)
- Checked shopping list item checkmark icon

Muted/supporting colors (from established codebase):
- `#1A1410` — primary text (`--casa-text`)
- `#6B5B3E` — secondary text / subheadings (`--casa-text-muted`)
- `#9B8B70` — faint captions, metadata (`--casa-text-faint`)
- `#E8E0D0` — borders (`--casa-border`)
- `#F0EBE0` — tab strip background, category pill background
- `#C4B49A` — empty state icon fill (de-emphasised)
- `#FEF9EC` — icon badge background (warm amber tint)

Checked item de-emphasis (SHOP-05):
- Text: `#9B8B70` (`--casa-text-faint`) — one level lighter than normal body

- Do NOT move checked items to the bottom in Phase 2 (offline sync in Phase 5 complicates order state; visual de-emphasis is sufficient).

## Component Inventory

### Client-side surfaces

#### Dashboard: Upcoming visit card (enhanced)
- Slot in existing dashboard grid below stats.
- If no confirmed plan exists for next visit: show "Plan menu" CTA button (`#8B6914` fill) and date display.
- If confirmed plan exists: show recipe list summary (2–4 recipe titles, inline count badge).
- Card border: `1px solid #E8E0D0`, background `#FFFFFF`, radius `rounded-xl`.

#### Menu plan list page (`/menus`)
- Layout: vertical stack of plan cards, one per upcoming visit.
- Plan card: visit date (bold), status badge, recipe count, "Edit" link.
- Status badges:
  - `draft` — neutral (`#F0EBE0` bg, `#6B5B3E` text)
  - `confirmed` — blue (`#EFF6FF` bg, `#1D4ED8` text) — reuse existing `statusColors` from `visita/page.tsx`
- Empty state: dashed border card, CalendarDays icon (`#C4B49A`), heading + CTA.

#### Menu plan create/edit page (`/menus/[id]`)
- Full page (not drawer — confirmed by D-01).
- Sections in order: date input, recipe slots (2–4), rules warnings panel, action bar.
- **Focal point:** The recipe slots region is the primary focal point of this page. It occupies the largest visual area, sits centrally, and is the user's main task area. The "Add recipe" and "Suggest menu" buttons directly below the slots anchor attention there. The "Confirm menu" action bar is visually distinct at the bottom to signal task completion, but the recipe slots receive the eye first.
- Date input: plain `<input type="date">` via shadcn `Input` component. No calendar widget (D-03).
- Recipe slots: ordered list of selected recipes; each slot has a remove button (×, 20px, `#9B8B70`).
- "Add recipe" button: secondary style (outline, `#8B6914` border and text), opens picker modal.
- "Suggest menu" button: secondary style with Sparkles icon from lucide.
- Rules warnings panel: shown only when violations exist; amber background `#FFF7ED`, `#9A3412` text, ChevronDown to expand. Soft warning only — does not block confirmation (UX decision: real-time validation on each recipe change).
- Confirm button: primary (`#8B6914` fill, white text), full-width on mobile, right-aligned on desktop.

#### Recipe picker modal
- Uses shadcn `Dialog` — full-screen on mobile, centered (max-w-2xl) on desktop.
- Interior replicates `recipes/page.tsx` pattern: search `Input` + category `Tabs` + grid.
- Selection: tap/click toggles selection; selected recipes show checkmark badge overlay on card top-right corner.
- Counter in modal header: "3 of 4 selected" (amber text when at limit).
- "Confirm selection" button: disabled until 2+ recipes selected; enabled at 2–4; `#8B6914` fill when enabled.
- Loading skeleton: 6 × `h-24 rounded-xl animate-pulse` in `#E8E0D0`.

#### AI suggest menu loading state
- Replace recipe slot area with 2–4 skeleton cards (pulsing) while Claude API call in flight.
- After response: populate slots with suggested recipes, show rationale in a collapsed info box (`#EFF6FF` bg, InfoIcon, "Why these recipes?" expand).

### Cook-side surfaces

#### Cook visita page (extended)
- Existing page at `app/(cook)/visita/page.tsx` gains a "Lista de compras" link button below recipe list.
- Button style: outline with ChefHat icon or ShoppingCart icon, full-width, `#8B6914` border.

#### Cook shopping list page (`/lista`)
- Layout: `max-w-lg mx-auto px-4 py-6` — matches existing visita page pattern.
- Header: running count chip ("3 / 12 listos") — `#8B6914` text, `#FEF9EC` background, top-right of header.
- Sections: recipe ingredients grouped by category, then fridge staples section at bottom (SHOP-06).
- Section header: 12px uppercase tracking-wide `#9B8B70` text, `py-2 border-b border-[#E8E0D0]`.
- List item row: `min-h-12` (48px) flex row. Left: Checkbox (shadcn). Right: ingredient name (Spanish), quantity+unit in muted text.
- Checkbox: 24px, `accent-color: #8B6914`, checked state fills with accent color.
- Checked row: `opacity-50`, name text changes to `#9B8B70`.
- Ad-hoc item add (SHOP-07): floating "+" button fixed bottom-right (`bottom-24 right-4`), 56px circle, `#8B6914` fill. Tap opens inline text input that appends to list.
- Fridge staples section: same list pattern with `is_always_stock` visual indicator — small "Siempre en casa" label in `#C4B49A` below item name.

## Accessibility

- Shopping list checkboxes: `aria-label="{item name}, {quantity} {unit}"` — includes item detail for screen reader context.
- Recipe picker modal: focus trap within Dialog, `aria-labelledby` pointing to modal heading.
- Rules warnings: `role="alert"` on warning panel when violations are present.
- Running count: `aria-live="polite"` so screen readers announce changes when items are checked.
- Touch targets: all interactive elements in cook shopping list meet 48px minimum (SHOP-03).
- Floating add-item button: `aria-label="Add item to shopping list"` — required because the button renders a "+" icon with no visible text label.
- Recipe slot remove button (×): `aria-label="Remove {recipe name}"` — required because the button renders an icon-only × with no visible text label.
