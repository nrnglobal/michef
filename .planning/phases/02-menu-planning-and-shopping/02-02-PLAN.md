    **Create lib/consolidate-ingredients.ts** â€” Pure function:

    Export `consolidateIngredients(recipes: Array<{ id: string; ingredients: Ingredient[] }>): ConsolidatedItem[]`

    Where `ConsolidatedItem` is:
    ```typescript
    export interface ConsolidatedItem {
      ingredient_name_en: string
      ingredient_name_es: string
      quantity: number | null
      unit: string | null
      category: string
      source_recipe_ids: string[]
    }
    ```

    Algorithm:
    1. Build a Map keyed by `lowercase(trim(ingredient.name_en)) + '|' + lowercase(trim(ingredient.unit))` â€” merge key includes unit so "2 cups flour" and "100g flour" stay separate
    2. For each recipe's ingredients array, for each ingredient:
       - Compute merge key
       - If key exists in map: add `parseFloat(ingredient.quantity)` to running total (skip if NaN), append recipe.id to source_recipe_ids (if not already present)
       - If key not in map: create new entry with `parseFloat(quantity)` (null if NaN), unit (null if empty), category (default to `'Other'` if absent)
    3. Return array of ConsolidatedItem sorted by category then name_en

    Unit normalization aliases (apply before merge key):
    - `'cups'` -> `'cup'`, `'tbsp'` -> `'tablespoon'`, `'tsp'` -> `'teaspoon'`, `'pieces'` -> `'piece'`, `'cloves'` -> `'clove'`

    **Create lib/rules-engine.ts** â€” Pure client-side function:

    Export `validateMenuPlan(recipes: Recipe[], rules: CookingRule[], recentPlans: MenuPlan & { recipeIds: string[] }[]): RulesWarning[]`

    Where `RulesWarning` is:
    ```typescript
    export interface RulesWarning {
      ruleId: string
      ruleType: string
      message: string
      severity: 'warning'
    }
    ```

    No `'use server'` directive â€” this runs client-side.

    Rule type checks:
    - `'dietary'` / `'allergy'`: Check if any recipe's `tags` array or `protein_type` contains the `rule_definition.restriction` string (case-insensitive). If match: warning with `rule_definition.description`.
    - `'preference'`: Check if `rule_definition.preference` is satisfied (e.g., "prefer fish on Fridays" â€” check visit date day-of-week). Simple string match; warn if not met.
    - `'frequency'`: Count how many times each recipe appears in `recentPlans` (last 4). Warn if any recipe in current selection exceeds `rule_definition.max_frequency ?? 2`.
    - `'exclusion'`: Check if `rule_definition.excluded_pair` (array of 2 recipe IDs) are both present in the current selection. Warn if so.

    Filter only `is_active === true` rules before checking.

    ```typescript
    export async function POST(request: Request) {
      const { visitDate, currentRecipeIds } = await request.json()
      const supabase = await createClient()

      // 1. Fetch all active recipes
      const { data: allRecipes } = await supabase
        .from('recipes')
        .select('id, title_en, category, protein_type, tags')
        .eq('is_active', true)

      // 2. Fetch active cooking rules
      const { data: rules } = await supabase
        .from('cooking_rules')
        .select('rule_type, rule_definition, is_active')
        .eq('is_active', true)

      // 3. Fetch recent confirmed plans (last 4) with their recipe IDs
      const { data: recentPlans } = await supabase
        .from('menu_plans')
        .select('id, visit_date, menu_plan_items(recipe_id)')
        .eq('status', 'confirmed')
        .order('visit_date', { ascending: false })
    ```
