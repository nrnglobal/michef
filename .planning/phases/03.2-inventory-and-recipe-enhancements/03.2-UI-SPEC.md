## Color

All color values come from the established `var(--casa-*)` custom property system defined in `globals.css`. Never use hardcoded hex values in new code — always reference these variables.

| Role | CSS Variable | Light Value | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | `var(--casa-bg)` | `#FAFAF8` | Page backgrounds, main content areas |
| Secondary (30%) | `var(--casa-surface)` / `var(--casa-surface-2)` | `#FFFFFF` / `#F5F0E8` | Cards, form panels, autocomplete dropdown background, Google Sheets import panel |
| Accent (10%) | `var(--casa-primary)` | `#8B6914` | Translate button (active/hover), autocomplete selected suggestion highlight, "Import from Sheets" primary CTA, file upload button |
| Destructive | `var(--casa-diff-del-text)` + `var(--casa-diff-del-bg)` | `#991B1B` / `#FEF2F2` | Remove inventory item confirmation, discard/cancel destructive actions |

Accent reserved for:
1. Translate button active and hover state
2. "Import from Sheets" primary CTA button
3. Selected suggestion row highlight in autocomplete dropdown
4. Confirmed "Add to Menu" button when menu is already confirmed (status badge accent)

Secondary semantic:
- Destructive actions only: use `var(--casa-diff-del-text)` on `var(--casa-diff-del-bg)` background. No dedicated `--casa-destructive` variable exists — this is the established pattern from Phase 03.1 (STATE.md decision: 2026-03-25).

Status badge colors for menu picker modal (D-02 from CONTEXT.md):
- Draft menus: `var(--casa-surface-3)` background, `var(--casa-text-muted)` text
- Confirmed menus: `var(--casa-primary-bg)` background, `var(--casa-primary-text)` text

Source: `globals.css` casa-* variables, recipe-card.tsx patterns, Phase 03.1 decisions from STATE.md.

## Interaction Contracts

### UX-10 — ItemForm focus fix

- The `ItemForm` component must be defined at module scope (outside `InventoryClient`) to prevent re-mount on every keystroke.
- No visible change to the user — this is a correctness fix. The form must retain focus through every onChange cycle.

### UX-05 — Ingredient autocomplete

- Trigger: shows after 1 character typed in the English name field.
- Max suggestions shown: 8.
- Debounce: none required — static JSON filter is synchronous and fast.
- Keyboard: ArrowDown/ArrowUp navigate suggestions; Enter or click selects; Escape closes.
- Selecting a suggestion: auto-fills Spanish name field and sets the category Select value. Both fields remain editable after auto-fill.
- No suggestion match: dropdown closes; user continues typing their custom item. No error state.
- Dropdown closes on: Escape key, click outside, blur of the name input, or suggestion selected.

### UX-09 — Translate button

- Location: inline to the right of the Spanish name field in ItemForm.
- Icon: `Languages` (already imported in `recipe-form.tsx`).
- States: idle → loading (spinner, disabled) → success (Spanish field populated, button returns to idle).
- Loading text: "Translating..." (screen reader only; button shows spinner icon).
- Only enabled when: English name field is non-empty AND Spanish name field is empty or different from a prior translation.
- Error: inline toast via `sonner` — "Could not translate. Try again."

### UX-06 — Google Sheets import

- Entry point: "Import from Sheets" button below the inventory list header — same visual level as "Add Item".
- Step 1: URL input panel expands (collapsible, same pattern as `url-import-section.tsx`). Input label: "Google Sheets share URL". Helper text: "Sheet must be shared with 'Anyone with the link'. Download template."
- Template link: downloads `public/templates/inventory-template.csv`.
- Step 2: "Import" button triggers server-side fetch. Loading state: spinner + "Fetching sheet...".
- Step 3: Results panel shows parsed rows with counts: "Found 24 items. 3 duplicates detected."
- Duplicate resolution: each duplicate row shows item name + "Skip" / "Update" buttons inline. Non-duplicate rows are imported immediately on submit.
- Step 4: "Confirm Import" button processes all decisions. Success: sonner toast "22 items added, 1 updated, 2 skipped." Panel collapses.
- Error states: invalid URL → inline error below input "Not a valid Google Sheets URL"; fetch failure → inline error "Could not fetch sheet. Check sharing settings."; empty sheet → "No items found in this sheet."

### UX-07 — Recipe image upload

- Location: replaces the "Image URL" text input in `recipe-form.tsx`.
- Display when no image: a bordered upload zone (dashed border, `var(--casa-border)`) with camera icon and label "Upload photo or take picture". Clicking opens native file/camera picker.
- Display when image present (upload or existing URL): 160×120px preview thumbnail with "Change Photo" button overlay (ghost variant, small). A "Remove" button (destructive ghost, icon-sm) removes the image.
- Upload progress: the upload zone shows a progress indicator (spinner centered over the zone) while resizing and uploading. The form save button is disabled until upload completes if an upload is in progress.
- For existing recipes with external `image_url` (Spoonacular/URL import): thumbnail displays normally; "Change Photo" replaces it with an upload; "Remove" clears it.
- Camera capture: the file input uses `accept="image/*" capture="environment"` — no custom viewfinder.

### UX-08 — Add to any menu

- The "Add to Menu" modal already exists; the only visible change is that the menu list now includes confirmed menus alongside draft menus.
- Each menu row must show: visit date + status badge (Draft / Confirmed) + recipe count.
- No other interaction change required.
